Two diverging tracks after completing the core 19-project ladder. Choose based on career direction. Both start at roughly Month 12+ of the ladder.

## Track A: FPGA Verification via cocotb

**Career target**: FPGA engineering, ASIC verification, digital design  
**Salary context**: FPGA engineers average $175k; cocotb is used at Intel, Xilinx/AMD, ASIC startups  
**Prerequisites**: Basic Verilog/VHDL knowledge (EEE 120 Digital Design at ASU)

### Project 20A: cocotb Testbench for a Verilog Counter
**New Python concept**: cocotb coroutines (`async def`), `cocotb.start_soon()`, `cocotb.Clock()`, DUT signal interaction  
**EE concept**: RTL testbench methodology; clock generation; reset sequencing; pass/fail assertions on digital outputs  
**Output**: cocotb testbench for a simple synchronous up-counter in Verilog; tests: reset behavior, count increment, overflow wrap; runs in Icarus Verilog (free, open-source simulator)  
**Libraries**: cocotb, simulator: Icarus Verilog (`iverilog`, free)

```python
import cocotb
from cocotb.clock import Clock
from cocotb.triggers import RisingEdge, FallingEdge

@cocotb.test()
async def test_counter(dut):
    clock = Clock(dut.clk, 10, units="ns")   # 100 MHz
    cocotb.start_soon(clock.start())

    # Reset
    dut.rst.value = 1
    await RisingEdge(dut.clk)
    await RisingEdge(dut.clk)
    dut.rst.value = 0

    # Verify counting
    for expected in range(16):
        await RisingEdge(dut.clk)
        assert dut.count.value == expected, \
            f"Expected {expected}, got {dut.count.value}"
    
    cocotb.log.info("Test PASSED: counter counts correctly")
```

**Verilog DUT** (`counter.v`):
```verilog
module counter #(parameter WIDTH=4) (
    input clk, rst,
    output reg [WIDTH-1:0] count
);
    always @(posedge clk)
        if (rst) count <= 0;
        else     count <= count + 1;
endmodule
```

**Makefile** (`Makefile`):
```makefile
SIM = icarus
TOPLEVEL_LANG = verilog
VERILOG_SOURCES = $(PWD)/counter.v
TOPLEVEL = counter
MODULE = test_counter
include $(shell cocotb-config --makefiles)/Makefile.sim
```

Run: `make` → cocotb drives the DUT and reports PASS/FAIL.

**Teaches**: cocotb is replacing SystemVerilog UVM in some flows due to Python expressiveness. Knowing Python testbenches + a HDL simulator = hireable at FPGA/ASIC companies. This is the entry point to the $175k FPGA track.  
**GitHub deliverable**: `counter.v` + `test_counter.py` + `Makefile` + passing test output screenshot

### Project 20B: Double Pulse Test (DPT) Data Analyzer
**New Python concept**: `scipy.signal.find_peaks()` for event detection, integration with `np.trapz()`, Pandas multi-column analysis  
**EE concept**: Double pulse test (DPT) — industry-standard switching loss measurement; Eon = turn-on energy loss; Eoff = turn-off energy loss; Qrr = diode reverse recovery charge; dV/dt and dI/dt during switching transients  
**Output**: Reads oscilloscope CSV from a double pulse test (or uses synthetic test data generated by Project 13); detects turn-on and turn-off events using peak detection on dV/dt; integrates V_ds × I_d to compute Eon and Eoff in μJ; plots switching waveform with event annotations  
**Libraries**: Pandas, SciPy, NumPy, Matplotlib

```python
import pandas as pd
import numpy as np
from scipy.signal import find_peaks
from scipy.integrate import trapz

df = pd.read_csv('dpt_waveform.csv')
t   = df['time_s'].values
Vds = df['Vds_V'].values
Id  = df['Id_A'].values

# Detect switching events via dV/dt threshold
dVdt = np.gradient(Vds, t)
turnon_idx, _  = find_peaks(-dVdt, height=1e6)   # V falls at turn-on
turnoff_idx, _ = find_peaks( dVdt, height=1e6)   # V rises at turn-off

# Compute Eon (integration window around turn-on event)
ton_start = turnon_idx[0] - 50
ton_end   = turnon_idx[0] + 200
Eon_uJ    = trapz(Vds[ton_start:ton_end] * Id[ton_start:ton_end],
                  t[ton_start:ton_end]) * 1e6  # convert J→μJ

# Compute Qrr from reverse recovery current
# Integrate negative Id current after turn-off
Qrr_nC = trapz(np.minimum(Id[turnoff_idx[0]:turnoff_idx[0]+300], 0),
               t[turnoff_idx[0]:turnoff_idx[0]+300]) * 1e9  # C→nC

print(f"Eon = {Eon_uJ:.2f} μJ")
print(f"Eoff ≈ {Eoff_uJ:.2f} μJ")
print(f"Qrr = {Qrr_nC:.1f} nC")
```

**Teaches**: The double pulse test is the gatekeeping question in WBG power electronics interviews. Being able to write a DPT analyzer demonstrates not just Python skill but deep understanding of switching losses — the core metric for SiC/GaN device evaluation. This project directly targets the LLNL/Wolfspeed/Tesla hiring filter.  
**GitHub deliverable**: `dpt_analyzer.py` + synthetic DPT waveform CSV + annotated switching plot + Eon/Eoff/Qrr output
